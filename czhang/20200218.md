## Notes

### this

    1. 隐式传递一个对象引用
    2. 两种误解：
       1. 函数对象本身
          解决方案1: => 使用函数对象的词法标识符来引用
          解决方案2: => 强制this的指向 foo.call(foo,i)
       2. 它的作用域 x (作用域存在于引擎内部，无法通过代码访问)
    3. this 的绑定和函数声明的位置无关，取决于执行上下文

### this 调用位置

      调用栈 中正在执行的函数的前一个调用

### this 绑定规则

1. 默认绑定
    1. 运行在严格模式下，全局对象不能默认绑定，会绑定到 undefined
    2. 严格模式下调用，不影响默认绑定
